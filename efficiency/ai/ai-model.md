# cursor v2.0.77模型对比分析

# 模型选择开关
- **Auto**：由 Cursor 根据任务自动选择模型，适合不想手动挑选时使用。
- **MAX Mode**：强制启用更大的上下文和算力，生成质量最高但更耗时、耗点。

## 单个模型定位
| 模型 | 定位 / 擅长 | 典型场景 |
| --- | --- | --- |
| Composer 1 | Anthropic 定制的全能型模型，写作、分析与一般代码都可 | 需求沟通、文档整理、需求分析 |
| Opus 4.5 (Claude) | Claude 的旗舰版，长上下文与复杂推理最强 | 架构设计、长篇对话、深度推理 |
| Sonnet 4.5 (Claude) | 速度与性能平衡版，比 Opus 快，推理稍弱 | 日常编码、代码评审、迭代开发 |
| GPT-5.1 Codex | OpenAI 面向代码定制版，读写多语言代码能力强 | 开发、重构、排查 Bug（当前默认） |
| GPT-5.1 | 通用大模型，创意写作与技术分析均衡 | 需要兼顾写作、方案设计与技术分析 |
| Gemini 3 Pro | Google 多模态旗舰，图像/跨语言理解优秀 | 解释截图、阅读 PDF、多语言问答 |
| GPT-5.1 Codex Mini | Codex 轻量版，响应快、成本低 | 小范围改动、快速交互、频繁试错 |
| Grok Code | X.ai 模型，偏脚本/网络风格，具最新资讯 | 写脚本、爬虫、需要网络热梗的输出 |
| Gemini 2.5 Flash | Google 高吞吐版，生成极快 | 批量摘要、快速 QA、简单结论类任务 |

## 使用建议
1. **写代码 / 查 Bug**：优先用 GPT-5.1 Codex；想更快可切 Codex Mini。
2. **复杂架构 / 深度讨论**：Opus 4.5 或 GPT-5.1（取决于偏好的厂商）。
3. **处理图像 / 混合内容**：Gemini 3 Pro。
4. **需要极速响应**：Gemini 2.5 Flash 或 Codex Mini。
5. **默认体验**：保持 Auto；关键任务再切换到 MAX Mode + 指定模型。

根据任务随时切换即可；不同面板也能使用不同模型，组合使用能兼顾质量与效率。
