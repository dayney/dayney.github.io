结束

读后感

在书中很多方法，在平时的开发中已经使用了，比喻建立hash表，多种状态的时候用对象中的属性查找，速度会更快些，另外，也便于维护，不修改业务逻辑的代码；

const OPTIONOBJ = {
    'qq': function(){
        console.log('qqqqq');
    },
    'ww': function(){
        console.log('wwwww');
    },
    'ee': function(){
        console.log('eeeee');
    },
    'rr': function(){
        console.log('rrrrr');
    },
    'tt': function(){
        console.log('ttttt');
    },
}

// 后台给的状态
OPTIONOBJ[status]();   // 通过这样的方式来调用，相对以前的if/ if...else/ switch等会更易于维护

平时做活动或者时段性的高访问量，可以使用CDN，临时增加带宽。
